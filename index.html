자동화 가이드

Make × Google Drive 연결 오류 해결 가이드

restricted scopes / redirect_uri_mismatch 오류를 단계별로 해결합니다.

왜 이런 오류가 뜨나요?

최근 Google 보안 정책이 강화되면서, Make에서 Google Drive / Gmail 같은 기능을 사용할 때 단순 로그인만으로 연결되지 않는 경우가 생겼습니다.

특히 아래 기능은 제한됩니다.

  • Google Drive 파일 읽기/쓰기
  • Watch Files (실시간 감시)
  • Gmail 읽기
  • Google Sheets 자동화

그래서 Google Cloud Console에서 직접 OAuth 인증을 설정해줘야 합니다.

전체 단계

  1. Google Cloud Console 접속
  2. 새 프로젝트 만들기
  3. Google Drive API 활성화
  4. OAuth 동의 화면 설정
  5. 테스트 사용자 추가
  6. OAuth Client 만들기
  7. Redirect URI 추가
  8. Client ID / Secret 복사
  9. Make에서 연결
  10. Google 로그인
  11. 자주 발생하는 오류

1 Google Cloud Console 접속

아래 사이트에 접속해 Google 계정으로 로그인합니다.

https://console.cloud.google.com/

2 새 프로젝트 만들기

상단 프로젝트 선택 → 새 프로젝트

예시

  • 프로젝트 이름: Make Automation

생성 완료 후 해당 프로젝트를 선택합니다.

3 Google Drive API 활성화

왼쪽 메뉴 API 및 서비스 → 라이브러리

  1. Google Drive API 검색
  2. 클릭
  3. 사용 버튼 클릭
필요 시 Google Sheets API도 같은 방식으로 활성화하세요.

4 OAuth 동의 화면 설정

왼쪽 메뉴 API 및 서비스 → OAuth 동의 화면

User Type

  • 외부(External) 선택

앱 이름

  • Make

사용자 지원 이메일

  • 본인 이메일

개발자 연락처

  • 본인 이메일

입력 후 저장.

5 테스트 사용자 추가

OAuth 동의 화면 → 대상(Audience) 또는 테스트 사용자

아래 이메일을 추가합니다.

  • 본인 회사 이메일
  • 실제 사용할 이메일

예시

  • suyeon.oh@day1company.co.kr

입력 후 저장.

6 OAuth Client 만들기

왼쪽 메뉴 사용자 인증 정보

→ 사용자 인증 정보 만들기 → OAuth 클라이언트 ID

애플리케이션 유형

  • 웹 애플리케이션

이름

  • Make Connection

7 Redirect URI 추가 (중요)

승인된 리디렉션 URI 항목에 아래 주소를 정확히 입력합니다.

https://www.integromat.com/oauth/cb/google-restricted
⚠️ 오타 / 슬래시(/) 위치까지 정확히 일치해야 합니다. 한 글자만 달라도 redirect_uri_mismatch 오류가 발생합니다.

입력 후 저장.

8 Client ID / Secret 복사

생성이 완료되면 아래 2개 값을 복사합니다.

  • Client ID
  • Client Secret

9 Make에서 연결

  1. Make로 돌아가 Google Drive 모듈 추가
  2. Create a connection 클릭
  3. Advanced settings 토글 켜기
  4. 앞서 복사한 Client ID / Client Secret 붙여넣기

10 Google 로그인

  1. Sign in with Google 클릭
  2. 요청된 권한 모두 허용
  3. 완료되면 연결 성공 ✅

자주 발생하는 오류

1. redirect_uri_mismatch

원인

  • Redirect URI 오타
  • 마지막 슬래시 누락 또는 잘못된 경로

해결 — 아래 주소를 그대로 등록하세요.

https://www.integromat.com/oauth/cb/google-restricted

2. restricted scopes 오류

원인

  • 테스트 사용자 미등록
  • OAuth 동의 화면 설정 누락
  • Gmail 개인 계정 사용

해결

  • 테스트 사용자 추가 (Step 5)
  • External 설정 확인 (Step 4)
  • 가능하면 회사 계정 사용

3. Access blocked

원인 — Google Drive API 미활성화

해결

  1. 왼쪽 메뉴 API 및 서비스 → 라이브러리
  2. Google Drive API 검색
  3. 클릭 후 사용 버튼 클릭
