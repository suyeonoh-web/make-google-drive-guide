자동화 가이드

Make × Google Drive 연결 오류 해결 가이드

실제 화면 스크린샷으로 따라가는 단계별 설정 가이드. restricted scopes / redirect_uri_mismatch 오류를 해결합니다.

왜 이런 오류가 뜨나요?

최근 Google 보안 정책이 강화되면서, Make에서 Google Drive / Gmail 같은 기능을 사용할 때 단순 로그인만으로 연결되지 않는 경우가 생겼습니다.

특히 아래 기능은 제한됩니다.

  • Google Drive 파일 읽기/쓰기
  • Watch Files (실시간 감시)
  • Gmail 읽기
  • Google Sheets 자동화

그래서 Google Cloud Console에서 직접 OAuth 인증을 설정해줘야 합니다.

1 Google Cloud Console 접속

아래 사이트에 접속해 Google 계정으로 로그인합니다.

https://console.cloud.google.com/
Google Cloud 대시보드 — 상단 좌측에서 프로젝트를 선택할 수 있습니다.

2 프로젝트 선택 열기

상단의 프로젝트 선택 영역을 클릭해 프로젝트 목록을 엽니다. 오른쪽 위의 새 프로젝트 버튼을 누릅니다.

프로젝트 선택 모달 우측 상단의 새 프로젝트 클릭.

3 새 프로젝트 만들기

프로젝트 이름을 입력합니다. 예: MAKE (0514)처럼 알아보기 쉬운 이름.

프로젝트 이름은 추후 변경 가능하지만 프로젝트 ID는 변경할 수 없습니다.

4 프로젝트 이름 확정

특수문자/공백 없이 입력하면 ID도 깔끔하게 생성됩니다 (예: MAKE0514). 만들기 버튼을 누릅니다.

프로젝트 ID는 자동 생성되며 이후 변경 불가입니다.

5 Google 인증 플랫폼 시작

왼쪽 메뉴 API 및 서비스 → OAuth 동의 화면으로 이동하면 Google 인증 플랫폼 화면이 보입니다. 시작하기 버튼을 클릭합니다.

아직 구성되지 않음 상태 — 가운데 시작하기 클릭.

6 앱 정보 입력

앱 이름

  • MAKE

사용자 지원 이메일

  • 본인 이메일 선택

입력 후 다음.

1단계 앱 정보 — 사용자에게 동의를 요청할 때 표시되는 정보입니다.

7 대상(외부) 선택

외부(External)를 선택합니다. Google 계정이 있는 모든 테스트 사용자가 쓸 수 있도록 하는 옵션입니다.

개인 계정으로 만들 땐 외부를 선택해야 합니다.

8 연락처 정보 입력

Google에서 프로젝트 변경 사항을 알릴 때 사용하는 이메일을 입력합니다. 본인 이메일로.

3단계 연락처 정보 — 알림 수신용 이메일.

9 정책 동의 후 만들기

Google API 서비스 사용자 데이터 정책에 동의하는 체크박스를 켜고 계속 → 만들기를 누릅니다.

4단계 완료 — 정책 동의 후 만들기.

10 OAuth 개요 확인

만들기가 완료되면 OAuth 개요 화면이 열립니다. 이 화면에서 OAuth 클라이언트 만들기 버튼을 누를 수 있지만, 일단 먼저 테스트 사용자를 추가합니다.

"OAuth 구성 생성됨" 메시지가 뜨면 정상.

11 테스트 사용자 화면 열기

왼쪽 메뉴에서 대상 클릭. 테스트 사용자 영역의 + Add users 버튼을 확인합니다.

테스트 사용자가 비어 있는 초기 상태. + Add users 클릭.

12 테스트 사용자 추가

실제 Make 연결에 사용할 본인 이메일을 추가합니다. 추가 후 사용자 정보 영역에 이메일이 1개 등록됩니다.

테스트 사용자에 본인 이메일이 들어가야 OAuth 동의가 통과합니다.
⚠️ 여기에 등록 안 된 계정으로 로그인하면 access_denied / restricted scopes 오류가 납니다.

13 OAuth 클라이언트 만들기 시작

왼쪽 메뉴 클라이언트 이동. 상단 + 클라이언트 만들기 클릭.

OAuth 2.0 클라이언트 ID 목록이 비어 있는 상태 — 새로 만듭니다.

14 클라이언트 유형/이름 설정

애플리케이션 유형

  • 웹 애플리케이션

이름

  • MAKE
유형은 반드시 웹 애플리케이션으로 선택.

15 Redirect URI 추가 (가장 중요)

스크롤을 내려 승인된 리디렉션 URI 영역에서 + URI 추가를 누르고 아래 주소를 정확히 입력합니다.

https://www.integromat.com/oauth/cb/google-restricted
⚠️ 오타 / 슬래시 위치까지 정확히 일치해야 합니다. 한 글자만 달라도 redirect_uri_mismatch 오류가 발생합니다.

입력 후 만들기.

URI 1 칸에 https://www.integromat.com/oauth/cb/google-restricted.

16 Client ID / Secret 확인

생성 완료 모달에 클라이언트 ID와 클라이언트 보안 비밀번호가 표시됩니다. 이 2개를 안전한 곳에 복사해 두세요.

⚠️ 이 모달을 닫으면 더 이상 비밀번호를 보거나 다운로드할 수 없습니다 — 반드시 미리 복사.

17 API 라이브러리 열기

이제 Google Drive API를 활성화할 차례입니다. 왼쪽 햄버거 메뉴 API 및 서비스 → 라이브러리 클릭.

햄버거 메뉴 → API 및 서비스 → 라이브러리.

18 Google Drive API 검색

검색창에 google drive를 입력하면 관련 API 목록이 나옵니다. 첫 번째 Google Drive API 카드를 클릭합니다.

유사한 API들이 함께 뜨므로 정확히 Google Drive API(첫 번째)를 선택.

19 Google Drive API 사용

Google Drive API 상세 페이지에서 파란색 사용 버튼을 클릭합니다.

사용 버튼 클릭 — 이게 사실상 Google Cloud 쪽 마지막 작업입니다.

20 API 사용 설정 확인

측정항목 화면이 뜨면 활성화 완료. 상태: 사용 설정됨이 표시됩니다.

측정항목 페이지가 보이면 Drive API가 정상 활성화된 상태.

21 클라이언트 상세 다시 확인

왼쪽 메뉴 클라이언트 → 만든 OAuth 클라이언트 클릭. 우측에서 Client ID와 Client Secret을 다시 확인할 수 있습니다.

Redirect URI가 google-restricted로 들어가 있는지 마지막 점검.

22 Make에서 연결

  1. Make로 돌아가 Google Drive 모듈 추가
  2. Create a connection 클릭
  3. Advanced settings 토글 켜기
  4. 앞서 복사한 Client ID / Client Secret 붙여넣기
  5. Sign in with Google → 권한 모두 허용

완료되면 Google Drive 모듈이 정상 연결됩니다. ✅

자주 발생하는 오류

1. redirect_uri_mismatch

원인

  • Redirect URI 오타
  • 잘못된 경로 (google-restricted가 아닌 다른 값으로 등록)

해결 — 아래 주소를 그대로 등록하세요.

https://www.integromat.com/oauth/cb/google-restricted

2. restricted scopes 오류

원인

  • 테스트 사용자 미등록
  • OAuth 동의 화면 설정 누락
  • Gmail 개인 계정 사용 시 정책 제한

해결

  • 테스트 사용자 추가 (Step 11~12)
  • External 설정 확인 (Step 7)
  • 가능하면 회사 계정 사용

3. Access blocked

원인 — Google Drive API 미활성화

해결

  1. 왼쪽 메뉴 API 및 서비스 → 라이브러리
  2. Google Drive API 검색
  3. 클릭 후 사용 버튼 클릭
